# DocLink - Сервис телемедицины

Полнофункциональный сервис телемедицины с поддержкой видеоконсультаций, электронной медкарты и системой оплаты поинтами.

## Архитектура

- **Backend**: FastAPI (Python)
- **Frontend**: React + TypeScript
- **База данных**: PostgreSQL
- **Хранилище файлов**: MinIO (S3-совместимое)
- **Кэш**: Redis

## Текущий статус (ноябрь 2025)

- Личный кабинет пациентов и врачей работает на едином дизайн-системном фреймворке с поддержкой темы/языка.
- Видеокомнаты переведены на WebRTC с perfect-negotiation, буферизацией ICE-кандидатов, безопасным управлением медиа и файловым чат-обменом.
- Расписание поддерживает как админов, так и врачей (самостоятельное открытие/удаление слотов, авто-обновление доступности для пациентов).
- Профили пациентов/врачей связаны с регистрацией (FIO + ДР), включают загрузку аватаров, истории консультаций и EMR.
- Лэндинг и auth-страницы синхронизированы по UI с кабинетом; все тексты локализованы.

### Основные пользовательские сценарии

| Роль      | Возможности                                                                                              |
|-----------|-----------------------------------------------------------------------------------------------------------|
| Пациент   | Регистрация с ФИО/ДР, пополнение кошелька, выбор врача/слота, участие в видео, доступ к EMR и файлам.     |
| Врач      | Модерация админом, заполнение профиля, самостоятельное открытие расписания, ведение консультаций, EMR.    |
| Админ     | Модерация врачей, управление пользователями, начисление поинтов, ручные консультации, мониторинг транзакций. |

## Последние изменения

- **Видео-комнаты** (`frontend/src/pages/ConsultationRoom.*`, `frontend/src/components/Icons.tsx`, `backend/app/consultations/*`):
  - Perfect Negotiation, буферизация ICE, устранение StrictMode race-condition.
  - UI наподобие Teams: отдельные кнопки Leave/End, адаптивность, безопасные сообщения об ошибках медиа.
  - Файловый чат, связь с EMR, обработка ориентации камеры, иконки вместо emoji.
- **Расписание и бронирование** (`frontend/src/pages/SchedulePage.*`, `backend/app/admin/*`, `backend/app/schedule/service.py`):
  - Врачи открывают/удаляют слоты в своём ЛК; пациенты видят auto-refresh сетку.
  - Админы могут создавать/редактировать слоты в отдельном модальном окне; добавлены проверки пересечений и связи с консультациями.
- **Профили / Dashboard** (`frontend/src/pages/ProfilePage.*`, `DashboardPage.*`, `backend/app/users/*`, `backend/app/doctors/*`):
  - Реальные данные профиля (FIO, specialty, bio, цена), загрузка аватаров, история консультаций и EMR-модалка.
  - Dashboard показывает баланс, реальные консультации/слоты, приветствие по ФИО.
- **Онбординг / Landing** (`frontend/src/pages/LandingPage.*`, `AuthPages.css`, `RegisterPage.tsx`, `AuthContext.tsx`, `backend/app/auth/*`, `backend/app/common/models.py`):
  - Переработанный Hero с графикой, реальный preview секции, переключатели темы/языка.
  - Регистрация требует ФИО и дату рождения, данные автоматически попадают в соответствующие профили.
- **Инфраструктура** (`frontend/Dockerfile`, `docker-compose.yml`, `frontend/src/index.tsx`):
  - Перевод фронтенда на prod build (без dev WebSocket 3000), удалён StrictMode, снизились ошибки SSL/ERR_PROTOCOL.

## Быстрый старт

### Требования

- Docker и Docker Compose
- Python 3.11+ (для локальной разработки)
- Node.js 18+ (для локальной разработки)

### Запуск через Docker Compose

1. Клонируйте репозиторий или скопируйте проект

2. Создайте файл `.env` в директории `backend/`:

```bash
cd backend
cp .env.example .env
# Отредактируйте .env файл с вашими настройками
```

3. Запустите проект:

```bash
docker-compose up -d
```

4. Примените миграции:

```bash
docker-compose exec backend alembic upgrade head
```

5. Откройте в браузере:
   - Frontend: http://localhost:3000
   - Backend API: http://localhost:8000
   - API Документация: http://localhost:8000/api/docs
   - MinIO Console: http://localhost:9001 (minioadmin/minioadmin)

## Основные модули

### 1. Аутентификация (Auth)
- Регистрация пользователей
- Авторизация через JWT
- Обновление токенов
- Подтверждение email

### 2. Пользователи (Users)
- Профили пациентов
- Медицинские файлы
- Управление данными

### 3. Врачи (Doctors)
- Профили врачей
- Сертификаты и документы
- Верификация администратором

### 4. Расписание (Schedule)
- Создание слотов расписания
- Резервация слотов
- Поиск доступных слотов

### 5. Консультации (Consultations)
- Создание консультаций
- WebRTC видеосвязь
- Текстовый чат
- Обмен файлами
- Управление статусами

### 6. Кошелек (Wallet)
- Баланс поинтов
- История транзакций
- Заморозка/разморозка поинтов

### 7. Платежи (Payments)
- Интеграция с Stripe
- Покупка поинтов
- Поддержка валют (RUB, USD, EUR)

### 8. ЭМК (EMR)
- Создание записей
- История консультаций
- Доступ по ролям

### 9. Админ-панель (Admin)
- Управление пользователями
- Модерация врачей
- Статистика
- Настройка курсов обмена

## Разработка

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

### Frontend

```bash
cd frontend
npm install
npm start
```

### Миграции

```bash
# Создать новую миграцию
alembic revision --autogenerate -m "Description"

# Применить миграции
alembic upgrade head

# Откатить миграцию
alembic downgrade -1
```

## Переменные окружения

Основные переменные окружения (см. `backend/.env.example`):

- `DATABASE_URL` - URL базы данных PostgreSQL
- `SECRET_KEY` - Секретный ключ для JWT
- `S3_ENDPOINT_URL` - URL MinIO/S3
- `STRIPE_SECRET_KEY` - Ключ Stripe API
- `POINTS_EXCHANGE_RATE_RUB/USD/EUR` - Курсы обмена валют
